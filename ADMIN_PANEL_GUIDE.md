# 🔧 Руководство по панели администратора

## 🚀 Первый запуск

### Шаг 1: Инициализация базы данных

Перед использованием панели администратора необходимо **один раз** инициализировать базу данных.

**Как это сделать:**

1. Откройте любой HTTP клиент (например, браузер, Postman или curl)

2. Выполните POST запрос на endpoint:
   ```
   POST https://[ваш-project-id].supabase.co/functions/v1/make-server-de695671/seed
   ```

3. **Через браузер** (самый простой способ):
   
   Откройте консоль браузера (F12) и выполните:
   ```javascript
   fetch('https://[ваш-project-id].supabase.co/functions/v1/make-server-de695671/seed', {
     method: 'POST',
     headers: {
       'Authorization': 'Bearer [ваш-anon-key]'
     }
   })
   .then(r => r.json())
   .then(console.log);
   ```

4. **Через curl**:
   ```bash
   curl -X POST https://[ваш-project-id].supabase.co/functions/v1/make-server-de695671/seed \
     -H "Authorization: Bearer [ваш-anon-key]"
   ```

**Что произойдет:**
- В базу данных будут добавлены 4 тестовых врача
- В базу данных будут добавлены 6 услуг
- Вы увидите ответ: `{"success": true}`

**⚠️ Важно:** Запускайте seed только **один раз**. При повторном запуске данные будут обновлены.

## 👨‍⚕️ Управление врачами

### Добавление нового врача

1. **Откройте панель администратора**
   - Войдите в систему как администратор
   - Перейдите в раздел "Панель управления" в меню

2. **Нажмите кнопку "Добавить"** ("+") в разделе "Врачи"

3. **Заполните форму:**
   - **Вспомогательная информация:**
     - Имя: Полное имя врача (например, "Dr. Anna Liepiņa")
     - Опыт: Количество лет опыта работы
   
   - **Специальность** (на трех языках):
     - Latviešu: "Galvenā veterinārārste"
     - Русский: "Главный ветеринар"
     - English: "Chief Veterinarian"
   
   - **Описание** (на трех языках):
     - Latviešu: Краткое описание врача на латышском
     - Русский: Краткое описание врача на русском
     - English: Краткое описание врача на английском
   
   - **Фотография врача:**
     - Нажмите кнопку "Загрузить изображение" / "Augšupielādēt attēlu" / "Upload image"
     - Выберите файл изображения (JPEG, PNG или WebP)
     - Максимальный размер: 5MB
     - Рекомендуемые размеры: минимум 400x400 пикселей
     - После загрузки фотография отобразится в форме
     - При необходимости можно удалить фото кнопкой "Удалить" / "Noņemt" / "Remove"
   
   - **Активность:**
     - Установите флажок "Активен", чтобы врач отображался на сайте
     - Снимите флажок, чтобы скрыть врача с сайта

4. **Нажмите "Сохранить"**
   - Врач будет добавлен в базу данных
   - Фотография будет автоматически загружена в хранилище Supabase
   - Врач появится в списке врачей на сайте (если он активен)

### Редактирование врача

1. Найдите врача в списке
2. Нажмите кнопку "Редактировать" (иконка карандаша)
3. Измените необходимые поля:
   - Можно изменить любую информацию, включая фотографию
   - Для замены фотографии - загрузите новое изображение
   - Для удаления фотографии - нажмите "Удалить" под превью
4. Нажмите "Сохранить"

### Удаление врача

1. Найдите врача в списке
2. Нажмите кнопку "Удалить" (иконка корзины)
3. Подтвердите удаление
4. Врач будет полностью удален из базы данных

### Управление фотографиями врачей

**Требования к фотографиям:**
- Форматы: JPEG, JPG, PNG, WebP
- Максимальный размер: 5 MB
- Рекомендуемое разрешение: минимум 400x400 пикселей
- Рекомендуемое соотношение сторон: 1:1 (квадрат) или 4:5 (портрет)

**Хранение изображений:**
- Все фотографии загружаются в приватное хранилище Supabase
- Используются подписанные URL для безопасного доступа
- Изображения автоматически оптимизируются для быстрой загрузки

**Отображение на сайте:**
- Фотографии врачей отображаются на странице "Врачи"
- Если у врача нет фотографии, показывается стандартное изображение
- Фотографии также видны в списке врачей в админ-панели

## 🎯 Структура данных врача

Каждый врач в системе имеет следующие поля:

```typescript
{
  id: "doctor:1",                    // Уникальный ID
  name: "Dr. Anna Liepiņa",          // Имя врача
  
  specialty: {                        // Специальность (мультиязычная)
    lv: "Galvenā veterinārārste",
    ru: "Главный ветеринар",
    en: "Chief Veterinarian"
  },
  
  experience: 15,                     // Опыт работы в годах
  
  description: {                      // Описание (мультиязычное)
    lv: "Текст на латышском...",
    ru: "Текст на русском...",
    en: "Text in English..."
  },
  
  specializations: {                  // Дополнительные специализации
    lv: ["Ķirurģija", "Terapija"],
    ru: ["Хирургия", "Терапия"],
    en: ["Surgery", "Therapy"]
  },
  
  schedule: {                         // Расписание работы
    monday: ["09:00-18:00"],
    tuesday: ["09:00-18:00"],
    wednesday: ["09:00-18:00"],
    thursday: ["09:00-18:00"],
    friday: ["09:00-18:00"],
    saturday: ["10:00-16:00"],
    sunday: []                        // Выходной
  },
  
  active: true                        // Статус (активен/неактивен)
}
```

## ⚠️ Устранение неполадок

### Проблема: "Не могу войти в админ панель"

**Решение:**
1. Убедитесь, что вы вошли с email: `admin@vet.lv`
2. Пароль может быть любым (например `12345`)
3. После входа перейдите в Профиль → Админ панель

---

### Проблема: "Врачи не загружаются (пустой список)"

**Причина:** База данных не инициализирована

**Решение:**
1. Выполните seed запрос (см. "Шаг 1: Инициализация базы данных")
2. Обновите страницу

---

### Проблема: "Ошибка при сохранении врача"

**ИСПРАВЛЕНО:** Теперь панель работает в демо-режиме без авторизации. 

**Если проблема осталась:**
1. Проверьте консоль браузера (F12) → вкладка Console
2. Найдите сообщения начинающиеся с "Saving doctor:" или "Error saving doctor:"
3. Проверьте что все обязательные поля заполнены (имя, опыт, специальность на всех 3 языках)
4. Убедитесь что база инициализирована (запустили seed)
5. Попробуйте обновить страницу и повторить

---

### Проблема: "Врач не удаляется"

**Решение:**
1. Убедитесь, что вы вошли как администратор
2. Проверьте консоль браузера на ошибки
3. Попробуйте обновить страницу и повторить

## 💡 Советы и лучшие практики

### ✅ Рекомендуется:

1. **Заполняйте все языковые версии**
   - Это важно для корректной работы мультиязычного сайта
   - Пользователи увидят информацию на выбранном языке

2. **Используйте деактивацию вместо удаления**
   - Деактивированный врач не будет отображаться на сайте
   - Но данные останутся в системе для истории

3. **Указывайте реальный опыт работы**
   - Это влияет на доверие пациентов

4. **Пишите подробные описания**
   - Помогают пациентам выбрать подходящего специалиста

### ❌ Не рекомендуется:

1. **Удалять врачей с активными записями**
   - Это может привести к проблемам с бронированием

2. **Оставлять пустые поля**
   - Особенно в многоязычных полях

3. **Использовать некорректные данные**
   - Опыт работы должен быть реальным числом

## 🔐 Безопасность

### Доступ к панели администратора

- Только пользователи с `isAdmin: true` могут управлять врачами
- Все операции требуют авторизации
- Токен доступа передается в заголовках запросов

### Рекомендации:

1. Не делитесь учетными данными администратора
2. Регулярно проверяйте список врачей
3. Используйте HTTPS в продакшн окружении

## 📞 API Endpoints

Для разработчиков - доступные endpoints:

```
GET    /doctors          - Получить всех врачей
GET    /doctors/:id      - Получить врача по ID
POST   /doctors          - Создать нового врача (auth)
PUT    /doctors/:id      - Обновить врача (auth)
DELETE /doctors/:id      - Удалить врача (auth)
```

**Пример запроса на создание врача:**

```javascript
fetch('https://[project-id].supabase.co/functions/v1/make-server-de695671/doctors', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer [access-token]'
  },
  body: JSON.stringify({
    name: 'Dr. New Doctor',
    specialty: { lv: '...', ru: '...', en: '...' },
    experience: 5,
    // ... другие поля
  })
});
```
